3. 统一约定

3.1 基础信息

基础路径 /api
数据格式 请求体与响应体均使用 JSON。GET 参数使用 query string。
金额与价格 后端返回多数字段为字符串形式的小数，如 "240.70"，下游展示时按 decimal 处理，避免浮点误差。
A 股数量规则 下单数量 quantity 必须大于 0，且必须为 100 股整数倍。
颜色约定 前端展示遵循中国惯例：红涨绿跌，即 text-up 为红色，text-down 为绿色。

3.2 统一响应结构

{
  "success": true,
  "data": {},
  "message": "ok"
}
字段 类型 说明
success boolean 业务是否成功。
data object | array | null 接口业务数据。不同接口结构不同。
message string 给调用方展示或记录的简短说明。

3.3 通用枚举

枚举 值 说明
account_type / mode live / paper / backtest 实盘 / 模拟盘 / 回测。当前同花顺桌面适配器下单只支持 live。
side buy / sell 买入 / 卖出。
scope today / history 查询当日委托/成交或历史委托/成交。
order_status submitted / partial_filled / filled / canceled / rejected / failed 订单生命周期状态。实盘状态以券商交易端返回为准。
source live / paper / backtest / replay / manual 数据来源。

POST /api/account/order

发起实盘限价委托。当前桌面适配器只实现 live 实盘模式，调用方必须传入幂等键，防止重复下单。

请求体

字段类型必填默认值约束说明
symbolstring是-长度 6-12证券代码。当前适配器可传 301183 等交易端识别格式；跨模块建议统一为 301183.SZ 后再由适配层转换。
sidestring是-buy / sell买卖方向。
pricedecimal是-> 0限价委托价格。
quantityinteger是-> 0，100 股整数倍委托股数。
modestring否live当前只支持 live账户模式。
idempotency_keystring实盘必填null最长 80幂等键，建议由调用方生成 UUID。
remarkstring否null最长 200人工备注或策略说明。
wait_manual_captchaboolean否true-遇到验证码时是否等待人工处理。
manual_captcha_timeoutinteger否18030-600等待人工验证码超时时间，单位秒。
 
{

  "side": "buy",
  "price": 240.70,
  "quantity": 100,
  "mode": "live",
  "idempotency_key": "9f972f9f-0d6d-46d0-a480-18c76a8c4c48",
  "remark": "strategy-signal-001",

响应 data

交易端返回的下单结果，通常包含请求回显、合同编号、可能成交记录、下单后资金等。最终状态以交易端委托列表和成交回报为准。

重要：调用 POST /api/account/order 成功只代表自动化委托流程执行完成，不等同于订单已全部成交。下游应继续调用 /api/account/orders 和 /api/account/trades 回查最终状态。

9. 错误码与异常

HTTP 状态码 场景 响应示例 下游处理建议
400 请求参数不符合业务规则，例如非 live 模式下单。 {"detail":{"message":"当前桌面适配器只实现 live 实盘模式。"}} 调用方修正参数后重试。
409 需要人工验证码，或交易客户端未就绪。 {"detail":{"message":"检测到验证码弹窗，请在同花顺中人工处理后重试。"}} 前端提示人工处理；策略执行应暂停自动下单并告警。
422 Pydantic 参数校验失败，例如数量不是 100 股整数倍、价格小于等于 0。 FastAPI 默认校验错误结构。 调用方按字段错误提示修正。
500 自动化适配器、数据库或未知服务异常。 {"detail":{"message":"...", "type":"ExceptionName"}} 记录请求参数和错误详情，通知账户交易模块负责人排查。

10. 推荐调用流程

策略执行下单流程

  1. 调用 GET /api/account/automation/status，确认 connected=true 且 ready=true。
  2. 调用 GET /api/account/balance 和 GET /api/account/positions，获取资金与持仓。
  3. 风控模块完成资金、仓位、涨跌停、停牌等检查。
  4. 调用 POST /api/account/order，必须携带 idempotency_key。
  5. 调用 GET /api/account/orders?scope=today 回查委托状态。
  6. 调用 GET /api/account/trades?scope=today 回查成交结果。

页面刷新同步流程

  1. 进入账户页面后调用 GET /api/account/automation/status。
  2. 用户点击同步时调用 POST /api/account/sync。
  3. 页面使用返回的 balance、positions、orders、trades 一次性刷新。
  4. 如果遇到 409 验证码，提示用户到同花顺客户端处理。

复盘分析流程

  1. 从账户交易模块读取成交、委托、持仓、资金快照。
  2. 按 account_type 区分实盘、模拟盘和回测数据。
  3. 生成复盘会话和指标快照。
  4. 下游调用 /api/review/report、/api/review/trades、/api/review/suggestions。

撤单流程

  1. 调用 GET /api/account/orders?scope=today 获取可撤委托。
  2. 取 broker_order_id 作为 entrust_no。
  3. 调用 POST /api/account/order/{entrust_no}/cancel。
  4. 再次调用 GET /api/account/orders?scope=today 确认状态。
